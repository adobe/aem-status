Github notifications unavailable

Postmortem

On 10.07.2025 18:25 UTC we received alerts that Operational Telemetry script delivery was failing.

This turned out to be a false positive, and no actual script delivery failures were observed.

Executive Summary

Nothing happened. Phew!

Impact Analysis

No customers or projects were impacted

Root Cause Analysis

Our monitoring for Operational Telemetry confirms that the scripts delivered by Operational Telemetry have checksums matching the published checksums on GitHub releases. This check uses the GitHub API to retrieve recent releases and release artifacts.

If the GitHub API is not available, or the worker running the check is rate limited, the check will fail. This has been causing the alert and our team was notified.

Next Steps

As this is a very infrequent occurrence and the cost of a false negative exceeds the cost of a false positive alert, no changes to the monitoring configuration will be made.
