# Pipeline timeout errors affecting Page Delivery

### Executive Summary

On Wednesday, April 8, 2026 between 15:01 UTC and 15:12 UTC, we observed a brief spike in 503 errors affecting AEM Delivery across multiple hosts. The incident was detected via automated error-rate monitoring. The errors were associated with the HTML pipeline and resulted in a small number of failed requests. The overall error rate remained well below 0.01% (~0.0068%), and impact to end users was minimal.

### Root Cause

The incident was caused by elevated backend latency in the HTML pipeline, leading to first byte and read timeouts. Requests exceeded expected response times, resulting in 503 responses being returned by the delivery layer. The issue was transient and affected multiple hosts, suggesting a short-lived backend performance degradation.

### Resolution

The issue resolved without manual intervention as backend latency returned to normal levels. No customer action is required.
